# Модель прецедентів

В цьому файлі необхідно перелічити всі документи, розроблені в проекті та дати посилання на них.

*Модель прецедентів повинна містити загальні оглядові діаграми та специфікації прецедентів.*



Вбудовування зображень діаграм здійснюється з використанням сервісу [plantuml.com](https://plantuml.com/). 

В markdown-файлі використовується опис діаграми

```md

<center style="
    border-radius:4px;
    border: 1px solid #cfd7e6;
    box-shadow: 0 1px 3px 0 rgba(89,105,129,.05), 0 1px 1px 0 rgba(0,0,0,.025);
    padding: 1em;"
>

@startuml

    right header
        <font size=24 color=black>Package: <b>UCD_3.0
    end header

    title
        <font size=18 color=black>UC_8. Редагувати конфігурацію порталу
        <font size=16 color=black>Діаграма прецедентів
    end title


    actor "Користувач" as User #eeeeaa
    
    package UCD_1{
        usecase "<b>UC_1</b>\nПереглянути список \nзвітів" as UC_1 #aaeeaa
    }
    
    usecase "<b>UC_1.1</b>\nЗастосувати фільтр" as UC_1.1
    usecase "<b>UC_1.2</b>\nПереглянути метадані \nзвіту" as UC_1.2  
    usecase "<b>UC_1.2.1</b>\nДати оцінку звіту" as UC_1.2.1  
    usecase "<b>UC_1.2.2</b>\nПереглянути інформацію \nпро авторів звіту" as UC_1.2.2
    
    package UCD_1 {
        usecase "<b>UC_4</b>\nВикликати звіт" as UC_4 #aaeeaa
    }
    
    usecase "<b>UC_1.1.1</b>\n Використати \nпошукові теги" as UC_1.1.1  
    usecase "<b>UC_1.1.2</b>\n Використати \nрядок пошуку" as UC_1.1.2
    usecase "<b>UC_1.1.3</b>\n Використати \nавторів" as UC_1.1.3  
    
    
    
    User -> UC_1
    UC_1.1 .u.> UC_1 :extends
    UC_1.2 .u.> UC_1 :extends
    UC_4 .d.> UC_1.2 :extends
    UC_1.2 .> UC_1.2 :extends
    UC_1.2.1 .u.> UC_1.2 :extends
    UC_1.2.2 .u.> UC_1.2 :extends
    UC_1 ..> UC_1.2.2 :extends
    
    
    UC_1.1.1 -u-|> UC_1.1
    UC_1.1.2 -u-|> UC_1.1
    UC_1.1.3 -u-|> UC_1.1
    
    right footer
        Аналітичний портал. Модель прецедентів.
        НТУУ КПІ ім.І.Сікорського
        Киів-2020
    end footer

@enduml

**Діаграма прецедентів**

</center>
```

### Завантаження нових даних

| **_ID:_**                | DATE_UPLOAD                                                         |
| ------------------------ | ------------------------------------------------------------------------------ |
| **_НАЗВА:_**             |  Завантаження нових даних                                                |
| **_УЧАСНИКИ:_**          | Адміністратор, Система                                                            |
| **_ПЕРЕДУМОВИ:_**        | Адміністратор ввійшов в систему                          |
| **_РЕЗУЛЬТАТ:_**         | Завантаження нових даних у базу даних (систему)              |
| **_ВИКЛЮЧНІ СИТУАЦІЇ:_** | Дані вже існують в системі (DATE_EXIST)        |

@startuml
|Адміністратор|
start;
: Входить в систему;
: Натискає кнопку "Завантажити дані";
: Обирає дані, які треба завантажити;

|Система|
: Перевіряє наявність цих даних у базі
<font color="red"><b> (Можлива помилка DATE_EXIST);
: Підтверджує, що цих даних немає в базі,\n дає дозвіл на продовження вибору даних;

|Адміністратор|
: Натискає кнопку "Завантажити дані";

|Система|
: Завантажує нові дані у базу даних;
stop;
@enduml

### Видалення даних

| **_ID:_**                | DATE_DELETE                                                         |
| ------------------------ | ------------------------------------------------------------------------------ |
| **_НАЗВА:_**             |  Видалення даних                                                |
| **_УЧАСНИКИ:_**          | Адміністратор, Система                                                           |
| **_ПЕРЕДУМОВИ:_**        | Адміністратор ввійшов в систему                           |
| **_РЕЗУЛЬТАТ:_**         | Видалення старих даних з бази даних                  |
| **_ВИКЛЮЧНІ СИТУАЦІЇ:_** | Даних вже не існує (DATE_DOES_NOT_EXIST)        |

@startuml
|Адміністратор|
start;
: Входить в систему;
: Натискає кнопку "Видалити дані";
: Обирає дані, які треба видалити;

|Система|
: Перевіряє наявність цих даних у базі \n<font color="red"><b> (Можлива помилка DATE_DOES_NOT_EXIST);
: Підтверджує, що ці дані є в базі;

|Адміністратор|
: Натискає кнопку "Остаточно видалити дані";

|Система|
: Видаляє дані у базу даних;
stop;
@enduml

| **_ID:_**                | DATE_EDITOR                                                         |
| ------------------------ | ------------------------------------------------------------------------------ |
| **_НАЗВА:_**             |  Надання прав на редагування даних користувачу                                 |
| **_УЧАСНИКИ:_**          | Адміністратор, Система                                                           |
| **_ПЕРЕДУМОВИ:_**        | Адміністратор ввійшов в систему                          |
| **_РЕЗУЛЬТАТ:_**         | Користувач отримує права на редагування                  |
| **_ВИКЛЮЧНІ СИТУАЦІЇ:_** | Адміністратор не підтверджує права на редагування (NO_EDITING_RIGHTS)        |

@startuml
|Адміністратор|
start;
: Входить в систему;
: Заходить в список користувачів,\n які надали запит на редагування даних;

|Система|
: Видає список всіх запитів;

|Адміністратор|
: Обирає потрібного користувача;
: Натискає на кнопку "Надати права редактора" \n навпроти імені користувача \n<font color="red"><b> (Можлива помилка NO_EDITING_RIGHTS);

|Система|
: Надає доступ на редагування;

stop;
@enduml

### Втрата прав на редагування даних

| **_ID:_**                | DATE_DELETE_EDITOR                                                         |
| ------------------------ | ------------------------------------------------------------------------------ |
| **_НАЗВА:_**             |  Втрата прав на редагування даних                                 |
| **_УЧАСНИКИ:_**          | Адміністратор, Система                                                           |
| **_ПЕРЕДУМОВИ:_**        | Адміністратор ввійшов в систему                      |
| **_РЕЗУЛЬТАТ:_**         | Користувач втрачає права на редагуванняння                  |
| **_ВИКЛЮЧНІ СИТУАЦІЇ:_** | Цей користувач не може бути позбавлений прав на редагування (CAN_NOT_DELETE_EDITOR) |

@startuml
|Адміністратор|
start;
: Надає запит на пошук всіх редакторів;

|Система|
 : Видає список всіх редакторів;

|Адміністратор|
: Обирає потрібного редактора;
: Натискає кнопку "Забрати права редактора" \n навпроти імені користувача \n<font color="red"><b> (Можлива помилка CAN_NOT_DELETE_EDITOR);

|Система|
: Забирає доступ на редагування;
stop;
@enduml
